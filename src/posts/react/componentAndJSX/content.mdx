### 네이밍 규칙

폴더 - 카멜케이스
컴포넌트 파일 - 파스칼 케이스


**자주 변경되는 영역같은걸 컴포넌트화 시키는게 좋다.**

단 너무 많은 것들을 분리하면 컴포넌트의 값들을 전달하거나 할떄 귀찮을 수 있으니 적절한 단위로 분리하는게 좋다. 



###### 컴포넌트끼리는 부모 자식간의 관계를 가질 수 있고 종속성이 있다. 


###### JSX란?

자바스크립트 코드이지만 html 요소도 생성 가능한 js + html

**예시코드** (이런거 됨)
```jsx
const start_half = 
	  <div><h1>ㅎㅇ</h1></div>
```

jsx에서 쓰는 `<div></div>` 등과 같은 html 태그는 DOM 요소라기보다 React요소이다.
**가상돔(Virtual DOM)** 이라는 개념 

따라서 jsx파일을 브라우저에서 직접 해석이 불가능하고 babel이라는 도구가 jsx를 js로 변환해줌

**코드스니펫**
- VSCode 확장앱 `react snippet` 검색
- 대충 다운 수 많은거 설치
- `rafce` 입력하면 자동으로 react 문법으로 컴포넌트 뼈대 생성



**문법**
1.`jsx`의 `return()` 안에 html 태그는 반드시 닫는 기호가 있어야 한다. (`</div>`)

2 `return()` 안에 부모 태그는 하나만 있어야 한다. 
```jsx
// 안되는 예시
return(
	<div></div>
	<div></div>
)
// 이렇게 최상위 부모는 하나만 있어야됨, 
return(
	<div>
		<div></div>
	</div>
)
```

3.빈태그를 부모로 만든 다면 ? (태그 필요 없는 것들)
```jsx
return(
	<>
		<div></div>
		<div></div>
	</>
)
```


4.배열 같은것도 html에서 보여줄 수 있다. (배열의 형태는 아님)
```jsx
const arr = [1, 2, 3];
  return (
    <>
      <div>{arr.map((n) => n)}</div>
  </>
```

5.클래스속성은 `className` 으로 정의한다. (jsx 예약어에 class가 있기 때문)
```jsx
<div
	id='abc'
	className='abc'>
```

6.인라인 css
일단 jsx에서 스타일은 스타일 속성이 들어있는 `객체` 형태로 넣어준다. 
따라서 기존의 css 스타일 속성의 `케밥케이스`를 사용하지 않고(객체의 속성은 케밥못씀)
`카멜케이스`를 쓴다.

```jsx
<p style={{
	color: 'orange'
}}>인라인 스타일 적용</p>
```
중괄호를 두 번쓰는데 이는 `jsx` 에서 변수등을 사용할 때 필수적으로 들어가는 `1차 {}` , 그리고 객체 속성이니까 `2차 {}` 를 사용하는 것

7.css 객체로 미리 정의해서 사용
``` jsx
const pStyle = {
	color: 'red',
	fontSize: '20px'
}
return (
	<>
	  <p style={pStyle}>객체 스타일 적용</p>
	</>
)

```



여러가지 문법 전체 코드
```jsx
import React from 'react'

const App = () => {

  const NUMBER = 1000;
  const arr = [1, 2, 3];

  const pStyle = {
    color: 'red',
    fontSize: '20px'
  }

  return (
    <>
      <div>첫 번째 줄</div>
      <div>{NUMBER}</div>
      <div>{arr.map((n) => n)}</div>

      {/* id, class */}
      <div
        id="abc"
        className='abc'>
        클래스
      </div>

      {/* style */}
      <p style={{
        color: 'orange'
      }}>인라인 스타일 적용</p>

      <p style={pStyle}>객체 스타일 적용</p>

    </>
  )
}

export default App
```
