## (목차1) App router와 Pages Router

일반적으로 Next.js를 채택할 때 주요한 의사 결정중에 하나이다. 
1. Pages router를 쓸 것인가?
2. [App Router](https://nextjs.org/docs/app)를 쓸 것인가?


### (목차 1-1) 1. Pages router

`pages` 폴더에 원하는 페이지의 파일 이름을 생성한다. 

```
pages
  ├── about.jsx
  ├── index.jsx
  ├── team.jsx
```

### (목차 1-2) 2. App Router

파일 시스템 기반으로 app 폴더 내의 폴더명을 기반으로 자동 라우팅이 된다. 
해당 폴더내의 `page.tsx` 가 페이지 컴포넌트 처럼 동작한다. (이름을 정확하게 작성해야함 대소문자 구별)

<Callout type="warn">
1. `page.tsx` 파일명은 고정이다.
2. 반드시 소문자로 작성한다.
3. `export default`가 반드시 있어야 한다. (`default` 를 page 컴포넌트로 인식한다.)
</Callout>

---

## (목차2) 동적 라우팅

React 프로젝트에서는 동적 라우팅을 설정하기 위해 `RRD`에서 다음과 같이 작성했다.

```JSX
<Route path='/post:id' element=<Post/>>
```

Next 프로젝트에서는 다음과 같이 작성하면 된다.
`segment`가 되는 폴더 하위에 [id] 등의 형식의 이름으로 폴더를 하나 더 생성한다.
`[id]` 하위에 `page.tsx` 를 생성한다. 



주의 사항(or 마무리)
<Callout type="warn">
1. `[id]`
</Callout>
